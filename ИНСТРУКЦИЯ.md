# Инструкция по публикации на GitHub Pages

## Шаг 1: Создайте репозиторий на GitHub

1. Откройте https://github.com/new
2. Заполните:
   - **Repository name**: например, `tundra147-diary` или `147-dney-v-tundre`
   - **Description**: "Дневник Фёдора Овчинникова - 147 дней в тундре"
   - **Visibility**: Public (для бесплатного GitHub Pages)
   - **НЕ** создавайте README, .gitignore или лицензию (у нас уже есть)
3. Нажмите "Create repository"

## Шаг 2: Подключите локальный репозиторий к GitHub

После создания репозитория GitHub покажет инструкции. Выполните:

```bash
cd "/Users/m.apalishin/Desktop/ ⚒️ 147 дней в тундре – фильм Фёдора Овчинникова"

# Замените <ваш-username> и <имя-репозитория> на реальные значения
git remote add origin https://github.com/<ваш-username>/<имя-репозитория>.git
git branch -M main
git push -u origin main
```

Или используйте готовый скрипт:

```bash
./deploy.sh <имя-репозитория> <ваш-github-username>
```

Например:
```bash
./deploy.sh tundra147-diary apalishin
```

## Шаг 3: Настройте GitHub Pages

1. Откройте ваш репозиторий на GitHub
2. Перейдите в **Settings** (в правом верхнем углу репозитория)
3. В левом меню выберите **Pages**
4. В разделе **Source**:
   - Выберите **Deploy from a branch**
   - Branch: выберите **main**
   - Folder: выберите **/ (root)**
5. Нажмите **Save**

## Шаг 4: Ожидайте публикации

GitHub Pages обычно публикует сайт за 1-2 минуты. Вы увидите зеленый баннер с адресом вашего сайта.

Адрес будет: `https://<ваш-username>.github.io/<имя-репозитория>/`

## Что дальше?

- Любые изменения в коде можно загружать командой:
  ```bash
  git add .
  git commit -m "Описание изменений"
  git push
  ```
  
- GitHub Pages автоматически обновится через несколько минут

## Примечание

Если у вас возникли проблемы с авторизацией при `git push`, используйте Personal Access Token:
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Создайте токен с правами `repo`
3. Используйте токен как пароль при запросе GitHub
